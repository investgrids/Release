Overview

Executive Summary

EventIQ is an Indian stock-market intelligence platform that surfaces market-moving events and traces their cascading effects across sectors, companies, and macroeconomic indicators. It targets retail investors, research analysts, and market observers who need event-driven context — not just price data.

The project consists of a Next.js 15 SSR/CSR frontend and a FastAPI backend backed by SQLite (dev) or PostgreSQL (prod). The flagship differentiator is the Ripple Engine — an AI-generated force-directed dependency graph that shows how one event cascades through 4 depth levels. All AI calls use free-tier models only (OpenRouter, Groq, Cerebras, Gemini).

Design System

The project uses a self-described "Data-First Dark Minimalism" visual language: bg-[#0a0d16] / bg-[#080c14] backgrounds, rounded-xl cards, border border-white/[0.07] hairline borders, no backdrop-filter blur on data cards, and indigo (#6366f1) as the primary accent. The layout root is xl:grid-cols-[240px_1fr_260px] with a 1600px max-width.

Structure

Site Architecture

Monorepo layout: d:/IG/apps/web (Next.js) and d:/IG/apps/backend (FastAPI). Node modules sit at the root d:/IG/node_modules/.

ConcernApproach
RenderingMixed SSR + CSR. Dashboard and Ripple Hub use SSR (force-dynamic); event list, news, stocks, radar are "use client" with static fallback data that loads instantly then silently swaps for live API data.
Data fetchingFrontend fetches NEXT_PUBLIC_API_BASE_URL (default http://localhost:8000). SSR pages use Node fetch with 7 s timeout; CSR pages use browser fetch in useEffect.
Caching3-tier: (1) in-process dict TTL cache in market.py, (2) Redis (30 min dashboard TTL; 1 h ripple TTL), (3) SQLite persist for ripple graphs.
SchedulingAPScheduler jobs: news ingest (every 15 min), dashboard pre-warm (7 AM IST daily), opportunity seeding (daily).
Error handlingEvery CSR page ships a STATIC_* fallback array rendered on mount; replaced silently if API responds. SSR pages use try/catch with Suspense per section.
AuthNone — fully public, no login system.
AlertsAlertProvider context wraps the whole app; BreakingNewsAlert component polls /api/alerts for high-impact breaking events.

Ripple Engine Graph
GET /ripple/[id]
CSR + AI Implemented Flagship

ReactFlow v11 force-directed graph. Custom node types: event-center, sector, company, commodity, currency, policy, indicator. Animated bezier edges colour-coded by impact (positive=emerald, negative=rose, mixed=amber). MiniMap with impact-coloured nodes. Controls (zoom/fit). Background dot grid.

Graph View (ReactFlow canvas), Insights Panel (AI summary + beneficiaries + losers + policy watch + market implications), Node Legend, Edge Types table.

GET /api/ripple/event/{event_id} — 3-tier cache: memory (1 h) → DB (ripple_graphs table) → AI generate → fallback template. Returns 17 nodes / 18 edges typical. POST /api/ripple/scenario for what-if.

Multi-provider chain: OmniRoute → Groq → Cerebras → Gemini → OpenRouter free models. 5 fallback templates keyed by event type (geopolitical, energy, monetary, fiscal, generic). Prompt instructs AI to return strict JSON with nodes[] and edges[].

Graph height: min(640px, 70vw). Touch events: touchAction: none. Tab bar: overflow-x-auto. Layout: flex-col lg:flex-row. Sidebar: w-full lg:w-[300px].

What-if scenario input field accessible from graph page but no hub-level entry point. No ability to save/share graph permalinks. No export (PNG/SVG). Node click should show detail panel — unclear if implemented.

Ripple Engine (models/ripple.py)

TableKey ColumnsNotes
ripple_graphsid Int PK, event_id String(256), scenario_type, scenario_input, event_title, event_summary, event_impact, graph_data JSON, insights JSON, generated_atscenario_type: event | scenario | company | theme. DB-level cache for AI-generated graphs. Created fresh on first server startup.

Schema dual-storage issue: The events table stores sectors and companies in both flat JSON columns (sectors JSON, companies JSON) AND normalised junction tables (event_sectors, event_companies). Code checks junction tables first, falls back to JSON columns. This dual-write means data can be inconsistent, and queries must handle both shapes.

Ripple AI Fallback Templates

When all LLM providers fail, ripple_service.py selects one of 5 keyword-detected templates based on event title/summary:

geopolitical — defence/conflict keywords energy — crude/oil/opec keywords monetary — rbi/rate/repo keywords fiscal — budget/capex/government keywords generic — all other events

Current State

Known Limitations

  • Data delay: All market data via yfinance is delayed ~15 min. No real-time tick data source. NSE Live / Upstox / Zerodha Kite APIs are not integrated.
  • No authentication: Zero access control. All data and AI features are publicly accessible. No rate limiting on AI endpoints.
  • AI latency: Ripple graph generation takes 5–15s on a cold cache. No streaming progress indicator during generation.
  • SQLite in dev: SQLite does not support JSONB operators, full-text search, or concurrent writes at scale. JSON columns use cast() workaround for LIKE queries.
  • No Alembic migrations: Schema changes require dropping and recreating tables, losing all data. Not viable in production.
  • In-process scheduler: APScheduler runs inside the uvicorn process. A crash restarts the scheduler but loses job state. Not suitable for high-availability production.
  • Hardcoded company lists: Stock Explorer (18 companies), Trending (10 chips), Compare (30 companies) are all hardcoded arrays. Adding a new stock requires a code change and deploy.
  • NSE API blocked: NSE India's APIs require browser session cookies + anti-bot headers. Server-side fetch without a session manager fails. FII/DII and PCR data unavailable as a result.
  • Impact score bug: Backend stores 0–10 float; frontend compares against 90/75/55 thresholds. Events with impact_score=8.5 appear as "Low Impact" — effectively all real data shows as Low.
  • No watchlist / persistence: No user state. Bookmarks, watchlists, and alert preferences are non-functional stubs with no backend storage.
  • RSS news only: No structured news data with sentiment, full article text, or NLP tagging. Articles are headline + summary only.
  • Free-tier LLM limits: Free model rate limits can cause AI feature failures during high-traffic periods. The fallback chain reduces (but does not eliminate) this risk.
  • Calendar data is seeded: No live economic calendar feed. Events must be manually entered or scraped.
